Create your blog using Hugo and GitHub

Posted on Jan 2, 2024

Here is a step-by-step guide on how to create a blog using Hugo and GitHub:

1. Prerequisites:

  • GitHub Account: Create a GitHub account if you don’t have one already.
  • Hugo Installation: Install Hugo. The installation command for macOS using Homebrew is provided in the video: brew install hugo.

2. Create GitHub Repositories:

  • Create a repository for your blog’s source code: This repository will hold your blog’s content, themes, and configurations. Name it something descriptive, like “my_blog_source”.
  • Create a repository for your blog’s deployment: This repository will host the statically generated files for your blog. Name it according to GitHub Pages conventions: your-username.github.io.

3. Set up Local Development Environment:

  • Clone the source code repository: Clone the “my_blog_source” repository to your local machine.
  • Create a new Hugo site: Inside the cloned repository, create a new Hugo site using the command: hugo new site my_blog_source-site.
Congratulations! Your new Hugo site was created in C:\Py\GitHub\rawblog\rnrawblog.

Just a few more steps...

1. Change the current directory to C:\Py\GitHub\rawblog\rnrawblog.
2. Create or install a theme:
   - Create a new theme with the command "hugo new theme <THEMENAME>"
   - Or, install a theme from https://themes.gohugo.io/
3. Edit hugo.toml, setting the "theme" property to the theme name.
4. Create new content with the command "hugo new content <SECTIONNAME>\<FILENAME>.<FORMAT>".
5. Start the embedded web server with the command "hugo server --buildDrafts".

See documentation at https://gohugo.io/.
  • Choose a theme: Explore available Hugo themes (link provided in the video). Select a theme you like and clone it into the themes directory within your Hugo site.
git clone https://github.com/athul/archie.git
  • Configure the theme: Open the config.toml file in your Hugo site.
    • Set the theme parameter to the name of the chosen theme.
    • Add a title for your blog.
    • Set the baseURL to https://your-username.github.io/ (replace with your actual username).
    • Additional config to Archie theme to change the color

4. Preview Your Blog Locally:

  • Start the Hugo server: hugo server. This will generate the static site and provide a local URL to preview your blog.

5. Create a New Blog Post:

  • Use the command hugo new posts/my-first-post.md to create a new Markdown file for your blog post.
  • Write your blog post content in Markdown format.
  • Created posts and about folders and added markdown files

6. Add the Deployment Repository as a Submodule:

  • In your source code repository, add the deployment repository (your-username.github.io) as a submodule: git submodule add -b main git@github.com:your-username/your-username.github.io.git public. This creates a public folder that will hold the generated static files.
C:\Py\GitHub\rawblog\rnrawblog>git submodule add -b main https://github.com/rnayaksec/rnayaksec.github.io.git public
fatal: 'rnrawblog/public' already exists and is not a valid git repo

The public folder is created when you run the hugo server; so delete it and rerun this command

7. Generate and Deploy Static Files:

  • Build the Hugo site: hugo -t your-theme-name. This generates the static files and places them in the public folder.

**8. Publish Your Blog:**

*   GitHub Pages will automatically detect the changes in the `main` branch of your deployment repository and publish your blog. You can access your blog at `https://your-username.github.io/`.

Key Points:

  • The source code repository (my_blog_source) holds the dynamic content and configurations of your blog.
  • The deployment repository (your-username.github.io) holds the statically generated files that are served by GitHub Pages.
  • Using a submodule helps keep your source code and deployment repositories organized.
  • GitHub Pages automatically publishes your blog whenever you push changes to the main branch of your deployment repository.
